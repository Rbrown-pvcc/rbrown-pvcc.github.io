Von Neumann Computer Architecture – 4 Main Parts:

  1. Memory Unit: Stores data and instructions the computer needs.
  2. Arithmetic Logic Unit (ALU): Does math and logic work.
  3. Control Unit: Tells the computer what to do step by step.
  4. Input/Output: Handles communication between the computer and the outside world.

Von Neumann Architecture – 3 Key Ideas:

  1. Stored Program: Instructions and data are stored in memory.
  2. Sequential Execution: One instruction runs at a time.
  3. Binary Data: Everything is represented with 0s and 1s.

Computer Architecture Terms:

  1. ROM: Memory you can only read, not change.
  2. Memory Cell Size: How many bits one memory cell holds.
  3. Memory Cell Address: A number that tells where a cell is stored.
  4. Maximum Memory Size: The largest memory the computer can use.
  5. Register: Very small, fast memory inside the CPU.
  6. Fetch: When the CPU gets an instruction from memory.
  7. Store: When the CPU puts data back into memory.
  8. Memory Access Time: How long it takes to read or write data.
  9. DASD: A storage device that can jump straight to data (like a hard drive).
  10. ALU: Does calculations and logical decisions.
  11. Control Unit: Directs all CPU operations.

Chapter 6

Four things the interface between the computer and the hardware does:

  1. Provides a way for hardware and software to communicate.
  2. Translates software commands into hardware actions.
  3. Manages data flow between components.
  4. Controls input/output devices and memory operations.

Responsibilities of systems software:

  1. Manage computer hardware resources.
  2. Provide a user interface.
  3. Control and run applications.
  4. Maintain file and memory organization.

Terms:

  1. Systems Software: Programs that manage hardware and system operations.
  2. User Interface: Part of the OS users interact with (command line or GUI).
  3. Language Services: Tools that help write and run programs (compilers, assemblers).
  4. Memory Managers: Control memory use and allocation.
  5. Information Managers: Handle data storage and retrieval.
  6. I/O Managers: Manage input and output operations.
  7. Scheduler: Decides which tasks run and when.
  8. Utilities: Support programs for maintenance (like file management).
  9. Assembly Language: Low-level language close to machine code.

Functions of an Operating System:

  1. Process Management
  2. Memory Management
  3. File Management
  4. Device Management
  5. Security and Control
